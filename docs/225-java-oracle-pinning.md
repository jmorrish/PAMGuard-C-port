# Java oracle pinning

Date: 2026-07-23

## Decision

The authoritative PAMGuard Java oracle is:

- repository: `https://github.com/PAMGuard/PAMGuard.git`
- version: `2.02.18e`
- commit: `dca55c81ef6f1498a8a3b926c69e7182afb915ee`
- local checkout: `PAMGuard_Java/`

The machine-readable pin is `reference-tools/pamguard-oracle.json`. Reference
tooling must fail rather than silently generate evidence from a different
commit.

## Why the previous version claim changed

Earlier documentation named `2.02.18b`. A clean build of that revision exposed
that it could not have generated the checked-in Hann fixtures: its
`WindowFunction.hann` divides by `length - 1`, while the fixtures and C++ port
divide by `length`. PAMGuard changed that implementation on 2026-06-29, after
`2.02.18b` and before the first C++ port checkpoint.

The newer Java line was therefore selected as the authority. A clean
`2.02.18e` build succeeded, and a newly generated length-8 Hann fixture was
byte-for-byte identical to the checked-in fixture. The C++ checker reported
`max_abs_error=0` and `gain_error=0`.

## Baseline validation

The entire reference corpus was regenerated on 2026-07-23:

- all 44 fixture-generator entry points completed successfully;
- 57 of 58 fixtures were byte-for-byte identical to the pre-run corpus;
- the remaining `.psfx` difference was traced to classifier UUID defaults in
  the reference sample writer, not to mapped settings or maths;
- those sample UUIDs are now fixed, and two consecutive `.psfx` generations
  produced the same SHA-256:
  `2BA536ED36738FA4E80E8D2213F6BB652CF3110CBE32B3F6218AECFCE9A69D62`;
- the regenerated project converted to the same session JSON;
- the C++ build completed and all 82 CTests passed with the regenerated corpus.

The pinned Java source also completed Maven `package` with tests skipped,
producing a 430.2 MB runnable JAR. Its manifest names `pamguard.Pamguard`, and
`java -jar ... -help` exited successfully.

Run `reference-tools/scripts/regenerate-java-fixtures.ps1` to repeat the full
fixture pass and the project-fixture determinism check.

## Claim boundary

This pin now governs the complete checked-in fixture corpus. It does not claim
that every PAMGuard module has been ported, nor that fixtures cover every
configuration or edge case; those limits remain in the parity ledger.
Compatibility with any other PAMGuard version is a separate claim and needs
fixtures generated by that version.
