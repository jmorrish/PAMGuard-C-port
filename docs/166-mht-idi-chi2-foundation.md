# MHT IDI Chi2 Foundation

Date: 2026-07-06

## Purpose

The ledger's top remaining click-train priority is PAMGuard's modern MHT click train detector. Its core track-quality measure is the inter-detection-interval chi-squared variable (`clickTrainDetector...mhtvar.IDIChi2` over `SimpleChi2Var`), which scores how smoothly the IDI changes along a candidate track and is combined by `StandardMHTChi2` across variables. This slice ports that variable with a fixture generated by the real Java class — the first concrete step of MHT porting.

## Reference semantics ported

`cpp-engine/src/detectors/MhtIdiChi2.cpp`:

- Batch `calcChi2`: successive-IDI chi contributions `(idi2 - idi1)^2 / max(idi1 * error, minError)^2`, averaged over `unitCount - 2`.
- Streaming `updateChi2` state machine: out-of-track kernel steps return the held chi2 over `bitcount`; the first usable unit resets the state; the first IDI is recorded without contributing; sub-minimum IDIs (`minIDI`, default 0.5 ms) add `StandardMHTChi2Params.JUNK_TRACK_PENALTY` (2e7).
- Times follow `IDIManager.calcTime`: nanosecond differences with a millisecond fallback for out-of-order units, retaining the sample-rate multiply/divide round trip for floating-point fidelity. Correlation-corrected IDIs (`useCorrelation`) are not ported.
- Defaults match `IDIChi2Params`: error 0.2, minError 0.0005, minIDI 0.0005.

## Fixture

`MhtIdiChi2FixtureExporter.java` drives the **real** `IDIChi2` class (no PamController needed at construction); data units use the fixed-nanosecond `PamDataUnit` subclass and a fixed-rate `IDIManager` subclass that reproduces `calcTimeSR`'s two branches without the parent-data-block lookup. Six cases: steady and jittered batch `calcChi2`, steady/jittered/skip (`bitSet` exclusion) streaming updates, and a junk-penalty update with a 0.2 ms IDI.

`mht_idi_chi2_parity` mirrors the catalogue and compares chi2 within `1e-9` relative.

## Claim boundary

This ports one chi2 variable. The MHT kernel itself (hypothesis branching, `TrackBitSet` management, pruning, garbage collection), the other chi2 variables (amplitude, bearing, correlation, length, peak frequency), `StandardMHTChi2` combination with track-quality bonuses, and train formation remain unported. The click train tracker's ledger status is unchanged; this row starts the MHT column of that work.
