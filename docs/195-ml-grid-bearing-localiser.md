# ML Grid Bearing Localiser

Date: 2026-07-21

## Purpose

Ports `MLGridBearingLocaliser2` — the localiser PAMGuard's `BearingLocaliserSelector` actually picks for plane and volume sub-arrays. `docs/194` ported the selection and recorded that the engine substituted its LSQ localiser for this one. That substitution is now unnecessary for the maths; this document covers the localiser itself, and wiring it into the served path follows separately.

## Reference semantics ported

The algorithm is a maximum-likelihood grid search, not a least-squares fit:

1. Build a lookup table over a theta/phi grid holding, for every hydrophone pair, the delay a plane wave from that direction would produce and the uncertainty on that delay.
2. Score an observed delay set as a Gaussian log likelihood against the table.
3. Take the interpolated peak of the resulting surface, and read errors from its curvature.

Grid extents and peak-search wrapping follow the sub-array shape, which is why `docs/194`'s selector had to come first:

| Shape | Theta range | Phi range | Theta wrapping |
| --- | --- | --- | --- |
| Line | `[0, pi]` | single bin at 0 | none |
| Plane | `[-pi, pi]` | `[0, pi/2]` | wraps, step 2 |
| Volume | `[-pi, pi]` | `[-pi/2, pi/2]` | wraps, step 2 |

A plane array searches the **upper hemisphere only** — the reference has `-pi/2` commented out on that branch. A plane sub-array cannot distinguish up from down, so the range is halved rather than reporting an arbitrary one of the two mirror solutions.

Delay errors combine three independent terms in quadrature: the hydrophone position errors projected onto the bearing direction, the speed-of-sound error scaled by the baseline projection, and the timing error. A grid bin where the array is endfire to the source has a large position-error term and therefore contributes weakly, which is how the likelihood surface encodes array geometry.

### Details that only a fixture settles

- **`Math.toRadians` multiplies by a precomputed constant.** Writing the 3-degree default step as `3 * pi / 180` instead of `3 * (pi / 180)` lands one ulp away, and that is enough to move the `floor()` that sets the bin count: 121 theta bins instead of 120. The first attempt at this port had exactly that bug, and the fixture caught it on the first run.
- **The theta grid does not reach `+pi`.** 120 bins from `-pi` in 3-degree steps stop at 177 degrees, because `2*pi / toRadians(3)` evaluates to just under 120. The peak search still wraps with step 2, which assumes the last bin *is* the first — a half-bin inconsistency inherited from the reference and preserved.
- **The peak interpolation reads the integer peak bins in both dimensions.** `PeakSearch.interpolatedPeakSearch` computes the rounded bins once, up front, so the theta offset does not move the row the phi interpolation reads.
- **`simplePeakSearch` seeds its running maximum with `data[0][0]`** and only a strictly greater value displaces it, so the first bin of a tied ridge wins. This matters for the symmetric test cases, where whole ridges tie exactly.
- **`getErrors` clamps the theta bin and then reuses the clamped value** for the phi curvature, so a peak at the grid edge reads its errors from one bin in.

## Implementation

`localisation::MlGridBearingLocaliser` builds the table in its constructor and holds it, so repeated `localise` calls over one geometry only pay for the likelihood sweep.

Two pieces of PAMGuard infrastructure came along with it:

- **Jama's LU decomposition and solve**, ported rather than replaced with a 3x3 cofactor inverse, so the array-axis rotation frame rounds the way the reference does. `LSQBearingLocaliser` set the precedent with Jama's Householder QR (`docs/158`).
- **`PeakSearch`'s 2D interpolated peak**, including the wrap rules and the quirks listed above.

`PamVector.getPerpendicularVector` is transcribed with its structure intact: its first branch catches every vector in the xy plane, which makes a later `vector[2] == 0` branch unreachable. The two branches return different vectors, and the reachable one is not the tidy one, so the port keeps the shape rather than simplifying it.

## Validation

`ml_grid_bearing_parity` (new) replays `tests/fixtures/localisation/ml-grid-bearing.csv`, generated by `MlGridBearingFixtureExporter`. The exporter transcribes `prepare`/`fullGridSearch` — that class reaches `ArrayManager.getArrayManager()` and the current `PamArray` in its constructor, both `PamController`-coupled — but every numerically delicate leaf is the **real** class: `PamVector` for the vector maths, `Jama.Matrix.inverse()` for the rotation frame, and `PeakSearch` for the interpolated peak. The array shape and direction transcription is reached by reflection into `ArrayShapeFixtureExporter` so the two exporters cannot drift apart.

Eleven cases: plane arrays broadside, diagonal, endfire, and elevated; a plane case with speed-of-sound error and one with asymmetric per-hydrophone position errors; volume arrays with the source above, below, and off-axis with error terms; and line arrays broadside and oblique. Every theta, phi, and error value matches to **7e-17** — the ported inverse and peak search track the reference essentially bitwise — so the test's tolerance is set at `1e-13` to catch real drift rather than accommodate it.

The exported delays travel in the fixture rather than being recomputed by the C++ check, so a mistake in the plane-wave formula cannot appear on both sides and read as parity.

Three guards are also pinned: a co-located pair prepares no grid, a delay set of the wrong length is rejected rather than padded, and a geometry whose delay errors are all zero reports nothing.

Full CTest suite passes `69/69`.

## Claim boundary

This slice is the localiser and its parity. Service wiring followed immediately in `docs/196-grid-bearing-service-output.md`, which adds per-hydrophone coordinate errors to session config and serves the result as `gridBearing` at schema v17.

The zero-delay-error guard is a **deliberate deviation**. The reference divides by zero there, producing an all-negative-infinity surface and returning whichever bin the peak search seeded from — a bearing with no meaning. The port returns nothing instead. This is reachable only with zero position, speed-of-sound, and timing errors, which is a misconfiguration rather than a real array.

`getSeparationErrorVector`'s streamer-level error term is not modelled: PAMGuard adds each streamer's own error vector in quadrature when two hydrophones are on different streamers, and the engine has no streamer error configuration. For single-streamer arrays — every case in the fixture — the two agree exactly.

The reference's alternative search paths (`localiseByCrawl`, `localiseBySimplex`, `localiseByBisection`) are unported. `fullGridSearch` is the one `localise` calls; the crawl is dead but complete, and the other two are stubs that return null.

Time-varying arrays are out of scope for the same reason as `docs/193`: `resetArray` rebuilds the table when the hydrophone locator is changeable, and the engine has no changeable locators.
